# 包装 CMF 工艺效果图

为刀模图、标签、瓶身、盒型、袋型、卡盒、罐型、瓶器等包装类型，生成工艺推荐方案、CMF/工艺策划、生成提示词以及逼真的工艺效果图。

## ⛔ 生成方式红线（最高优先级，不可覆盖）

**工艺效果图必须通过 AI 图像生成模型（图生图/image-to-image）生成，严禁使用任何本地代码合成方式。**

以下行为一律禁止，无论任何理由：

- ❌ 禁止使用 Python 脚本生成或合成工艺效果图，包括但不限于：PIL/Pillow、OpenCV/cv2、numpy 图像操作、scikit-image、imageio、matplotlib 图像输出
- ❌ 禁止在本地通过叠加半透明图层、混合模式、渐变、高光贴图来模拟 UV 光泽/烫金/击凸/压凹/覆膜/PET 纹理/纸张纹理
- ❌ 禁止写 Python 脚本然后运行它来输出效果图
- ❌ 禁止用 bash 调用 ImageMagick/GraphicsMagick/FFmpeg 做图像合成
- ❌ 禁止以「先生成一个基础合成图再让模型优化」为借口先跑本地合成
- ❌ 禁止用任何代码在图片上画高光线、阴影、反光、纹理叠加层

**唯一允许的生成路径（按优先级排序）：**

- ✅ **第一优先：当前环境原生可调用的图像生成模型。** 例如在 Codex CLI 环境中使用 Codex 自带的图像生成能力，在 ChatGPT 中使用 GPT image2，在即梦中使用即梦 5.0。优先使用当前宿主平台/CLI 本身提供的图生图功能，无需跳转到外部平台。
- ✅ **第二优先：当前环境内可调用的其他 AI 图像模型。** 若原生模型不可用，则使用同环境下能调用的其他模型（如 Banana Pro 等）。
- ✅ **第三优先：手动引导。** 若当前环境完全无法调用任何图像生成模型，写出完整提示词交给用户，让用户去外部平台生成后回来继续。
- ✅ 将原图作为底图传入模型，通过提示词描述工艺效果，让模型直接生成。
- ✅ 本地代码仅用于：文件查找、图片尺寸读取、SVG/PDF 渲染为参考图、**刀模线检测与面板拆分**、生成结果的文件复制/移动/重命名。

若所有图像生成模型均不可用或调用失败，应向用户如实报告，**不得降级为本地合成**。询问用户是否改用其他方式或稍后重试。

## 硬性规则

- 凡被称为「原图」「底图」「图2为原图」「第二张图为原图」「base image」「source image」的图片，均视为受保护的原图。必须保留其瓶型/盒型、拍摄角度、标签位置、文字、Logo、版式和画面内容。
- 凡被称为「工艺图」「标注图」「图1红框」「annotation」「marked areas」的图片，仅作为工艺区域映射参考。不得将标注框、箭头、注释、红色外框、遮罩或选区标记渲染到最终效果图中。
- 本地代码仅用于：文件查找、尺寸检查、SVG/PDF 转参考图渲染、刀模线检测与面板拆分、生成结果复制/移动和验证。**本地代码拆分面板是为了给图像模型提供精确纹理输入，不是最终合成。**
- **每个 SVG/平面稿都是独立的新包装设计，必须独立分析。** 不得把上一个设计的信息（盒型、材质、工艺、参考图）带到下一个设计中。
- **不得使用不相关的 3D 效果图作为新包装的参考。** 除非用户明确指定了参考图且该参考图确实是同一产品。
- 生成前先确认或推断各图片的角色定位。若角色不明确，在继续之前简要说明你的判断。
- 不得重新设计包装。不得更改产品名称、品牌、比例、颜色、版式、文字层级、瓶型/盒型结构或拍摄视角，除非用户明确要求。
- 保护中文文字和小号法规文字。提示词中应要求图像模型保留原有字体排印，避免出现虚假、变形、拼写错误或随机生成的中文字符。
- **中文文字变形是当前所有图像生成模型的已知局限。** 图生图过程中中文文字极易出现变形、笔画错乱、形近字替换等问题。缓解策略：① 拆分面板让模型将原始文字作为纹理贴图而非重新生成；② 提示词中明确声明「保留原始面板中的中文文字，不重新生成文字」；③ 将每个面板作为精确纹理贴图；④ 若文字变形严重且多次迭代无效，如实向用户说明。不得隐瞒文字变形问题。
- 保持工艺效果物理合理：附着于包装表面、与光线方向一致、克制、可量产，不得出现浮空、发光、溢色或贴纸般的脱离感。
- 不得将工艺效果图冒充为印刷级生产文件。始终将视觉效果指导与印前要求区分开。
- 当图片信息不清晰时，不得凭空编造尺寸、纸张克重、刀模具体含义或生产确定性。应明确指出不确定性，并建议与印刷供应商确认。
- 将刀模/切割/压痕/折叠引导线视为结构元数据，而非印刷内容。除非用户明确说明这些线条属于印刷设计的一部分，否则不得在成品包装上渲染黑色、蓝色、红色或其他颜色的引导线。

## 必须执行的工作流程

**⚡ 核心原则：用户发文件过来就是要看效果图的。分析→拆面板→生成效果图是一个不可中断的流水线。不要在中间停下来只输出文档。**

### 1. 首先分析输入内容

- 识别每张图片的角色：受保护原图/底图、工艺标注图、平面稿、刀模图、3D 效果图、材质参考、工艺参考或造型参考。
- **若输入是全新的 SVG/平面稿，它就是受保护原图。** 不要用其他图片混淆它的角色。
- 若输入为 SVG/PDF/AI 或其他矢量类刀模文件，优先进行矢量预处理：
  - 读取 viewBox/页面尺寸，提取画板宽高和比例
  - 识别切割/压痕/折叠/描边颜色（蓝色=折线 cls-2，黑色=切割线 cls-3 等）
  - 渲染一张高分辨率参考 PNG（2x 分辨率）
- 识别包装结构：瓶身、标签、盒型、袋型、卡盒、罐型、瓶盖、接缝、折叠、圆角、可见正面、侧面或环绕区域。
- 对于刀模图或平面结构，识别可能的盒型：天地盖盒、反向插锁盒、插口盒、锁底盒、飞机盒、抽屉盒、书型盒、手提盒、异形盒、纸袋、瓶标、收缩套标、袋型，或标注不确定。
- 对于刀模图，尽可能识别可见元素：切割线、压痕/折叠线、糊口、插舌、盖面板、底面板、底部结构、窗口、挂孔、出血、安全边距、正面板、侧面板、背面板。
- 识别文字敏感区域：中文标题、Logo、品牌名、条形码、二维码、营养成分表、配料表、法规文案、用法用量说明、小号文字。
- 识别可能的工艺区域：Logo、主视觉、产品名、图标、图案、边框、背景、装饰线条、卖点标识。
- 若刀模/结构不清晰，如实说明，避免过度自信的结构判断。

### 2. 仅澄清影响物理正确性的问题

- **若用户已经提供了图片文件、材质/工艺需求、输出位置，直接进入步骤 3→4→5 生成效果图，不要停下来问问题，不要只输出文档。**
- 若没有工艺标注图或明确工艺指令，询问工艺区域。
- 若未说明目标，默认视为视觉效果方案，直接生成效果图。
- 当产品品类、定位诉求、品牌风格会实质性影响材质/工艺推荐时，主动询问。
- 若用户已提供材质/工艺，直接继续，无需额外提问。
- 若用户要求推荐但未指定材质/工艺，提供简洁的多选选项后直接按用户选择生成。使用 `references/selection-questions.md`。

### 3. 生成提示词前先规划工艺

- 映射 区域 → 材质/工艺 → 强度。
- 主要工艺控制在 1–3 种，除非用户要求展示级综合方案。
- 信息面板和小号文字区域尽量不做处理以保持可读性，除非用户要求。
- 根据结构类型应用相应推荐。选择盒型材质/工艺时参考 `references/structure-recommendations.md`。

### 4. 输出顾问级交付物（与生成并行，不阻塞出图）

- **此步骤和步骤 5 同时进行，不要等文档写完再出图。** 先启动图生图（步骤 5），在等待生成结果时补充 CMF 方案说明。
- 提供图片识别结果、推荐材质方案、工艺分布表、效果图提示词、反向提示词、印前注意事项和交付文件建议。
- 使用 `references/output-format.md` 作为输出结构规范。
- 若用户仅需要提示词/方案策划，生成完提示词和工艺方案后即停止，不生成图片。

### 5. ⭐ 拆面板 + 生成（核心流水线，不可跳过任何一步）

**⚠️ 如果跳过拆面板直接把整张平铺图传给模型，结果必然失败——内容会变、比例会错、文字会乱码。拆面板不是可选策略，是强制步骤。**

**5a. 渲染 SVG 为平铺 PNG**
```bash
python3 -c "
import ctypes.util, os, sys
_orig = ctypes.util.find_library
def _patched(n):
    for p in [f'/opt/homebrew/lib/lib{n}.dylib', f'/opt/homebrew/lib/{n}.dylib',
              f'/opt/homebrew/lib/lib{n}.2.dylib']:
        if os.path.exists(p): return p
    return _orig(n)
ctypes.util.find_library = _patched
import cairosvg
cairosvg.svg2png(url='SVG文件路径', write_to='/tmp/flat_render.png', output_width=W, output_height=H)
print(f'Rendered: {os.path.getsize(\"/tmp/flat_render.png\")} bytes')
"
```

**5b. 检测刀模网格 + 拆分面板图片 + 制作标注图**
```bash
python3 << 'PYEOF'
import cv2, numpy as np, os

img = cv2.imread('/tmp/flat_render.png')
H, W = img.shape[:2]

# 检测蓝色刀模线
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
mask_blue = cv2.inRange(hsv, np.array([100,50,50]), np.array([130,255,255]))
lines = cv2.HoughLinesP(mask_blue, 1, np.pi/180, threshold=100, minLineLength=100, maxLineGap=10)

h_vals, v_vals = [], []
if lines is not None:
    for line in lines:
        x1,y1,x2,y2 = line[0]
        if abs(y2-y1) < 10: h_vals.append((y1+y2)//2)
        elif abs(x2-x1) < 10: v_vals.append((x1+x2)//2)

# 聚类
def cluster(vals, thresh=50):
    if not vals: return []
    vals = sorted(vals)
    clusters, cur = [], [vals[0]]
    for v in vals[1:]:
        if v - cur[-1] < thresh: cur.append(v)
        else: clusters.append(int(np.mean(cur))); cur = [v]
    clusters.append(int(np.mean(cur)))
    return clusters

h_lines = cluster(h_vals, 60)
v_lines = cluster(v_vals, 60)

# 如果检测不到足够线，用等分法回退
if len(h_lines) < 2 or len(v_lines) < 2:
    # 按 SVG viewBox 比例估算网格
    # 常见布局：2-4 行 × 2-4 列
    h_lines = [int(H*i/4) for i in range(5)]  # 4行
    v_lines = [int(W*i/4) for i in range(5)]  # 4列

print(f"Grid: {len(h_lines)-1} rows x {len(v_lines)-1} cols")
print(f"H lines: {h_lines}")
print(f"V lines: {v_lines}")

out_dir = '/tmp/box_panels'
os.makedirs(out_dir, exist_ok=True)

# 制作标注图
diagram = img.copy()
font = cv2.FONT_HERSHEY_SIMPLEX
labels = 'ABCDEFGH'

panel_idx = 0
for row in range(len(h_lines)-1):
    for col in range(len(v_lines)-1):
        x1, x2 = v_lines[col], v_lines[col+1]
        y1, y2 = h_lines[row], h_lines[row+1]
        panel = img[y1:y2, x1:x2]
        if panel.shape[0] < 30 or panel.shape[1] < 30:
            continue
        
        # 过滤空白面板（内容极少）
        gray = cv2.cvtColor(panel, cv2.COLOR_BGR2GRAY)
        if gray.std() < 15:
            continue
        
        # 简短编号 A1, B2, C3 ...
        label = f"{labels[row]}{col+1}"
        fname = f'{out_dir}/{label}.png'
        cv2.imwrite(fname, panel)
        
        # 标注图：绿色粗线 + 白色大字
        cv2.rectangle(diagram, (x1,y1), (x2,y2), (0,255,0), 4)
        # 白色大文字
        (tw, th), _ = cv2.getTextSize(label, font, 2, 4)
        cv2.putText(diagram, label, (x1+(x2-x1-tw)//2, y1+(y2-y1+th)//2), font, 2, (255,255,255), 4)
        
        panel_idx += 1
        print(f"  {label}: {panel.shape[1]}x{panel.shape[0]} px")

cv2.imwrite(f'{out_dir}/00_diagram.png', diagram)
print(f"Total: {panel_idx} panels -> {out_dir}/")
PYEOF
```

**5c. 验证面板已拆分**
- 检查 `/tmp/box_panels/` 目录下是否有多个 `A1.png` `B1.png` ... 文件
- 检查 `00_diagram.png` 是否存在
- **如果没有这些文件，不要继续下一步。重新执行 5b。**

**5d. ⭐ 询问包装物理尺寸（防止比例错误的关键步骤）**

**图像模型无法从 pt 单位推断真实物理尺寸。必须向用户确认实际尺寸。**

拆完面板后，根据检测到的网格比例，向用户提问：

```
折叠后盒子的面板网格为 {rows}行×{cols}列，每格约 {W}×{H} pt（比例 {ratio}:1）。

请确认包装的实际物理尺寸：
- 宽度（正面）约 ? mm
- 高度约 ? mm  
- 深度（侧面宽度）约 ? mm
```

如果用户不知道或说「按常规来」，根据产品类型给默认值：
- 50g 坚果小包装 → 约 80×80×30mm
- 标准礼盒 → 约 250×200×80mm
- 化妆品小盒 → 约 60×60×25mm
- 食品彩盒 → 约 120×80×40mm

**将确认后的物理尺寸写入提示词**：「折叠后盒体约 W×H×D mm（宽×高×深），比例 W:H:D」

**5e. 将所有面板打包传给图像模型**
- 用 `codex exec -i` 传入每个面板图片 + 标注图
- 命令示例：
```bash
cd "输出目录" && printf '%s' '图生图提示词...' | codex exec --skip-git-repo-check \
  -i /tmp/box_panels/A1.png \
  -i /tmp/box_panels/A2.png \
  -i /tmp/box_panels/B1.png \
  -i /tmp/box_panels/B2.png \
  -i /tmp/box_panels/00_diagram.png \
  2>&1
```

**5f. 提示词必须包含**
- 面板编号和对应盒面：「A1=正面、A2=背面、B1=左侧面...」
- 面板 pt 尺寸 + **物理 mm 尺寸**：「每格约 X×Y pt，折叠后盒体约 W×H×D mm（宽×高×深），比例 W:H:D」
- 盒型结构描述 + 禁止特征
- 材质工艺描述
- 「保留每个面板的原始设计内容，不重新生成文字」
- 「蓝色刀模线和绿色标注框不出现在最终图中」
- 输出路径

**5g. 不得跳过拆面板和尺寸确认。** 如果没有拆面板就直接传整张平铺图，或者没有确认物理尺寸就生成，结果会是错误的。

### 6. 保存输出

- 内置图像生成功能将结果保存至 `$CODEX_HOME/generated_images`。将选定结果复制到用户指定的文件夹。
- 使用包含材质、工艺和版本的描述性文件名，例如 `产品名_盒型_材质工艺_v1.png`。
- 除非用户明确要求，否则不覆盖已有文件。

### 7. 最终回复前进行质量检查

- 检查标注框是否已消失。
- 检查原包装结构是否仍然可辨识。
- 检查文字/Logo/版式是否发生偏移。若偏移严重，应使用更强的保护约束重新生成，而非本地修补。
- 检查盒型是否正确（天地盖不像飞机盒，反向插锁盒不是硬纸板礼盒等）。
- 检查比例是否正确（是否跟面板尺寸一致）。
- 检查工艺效果是否保持在指定区域内，并跟随透视、曲面、折叠和光线方向。
- 对于基于刀模图的输出，提醒该渲染图不能替代 CAD/AI 刀模确认。

## 选择和推荐行为

当用户说「帮我判断」「适合什么工艺」「推荐」「这个盒型怎么做工艺」，或提供了不清晰的刀模图时：
- 首先分析图片并列出不确定因素。
- 然后提出少量多选问题，而非直接给出完整推荐。
- 若用户说「不确定，你推荐」，采用克制的中高端默认方案：根据结构匹配合适基材、哑光/软触感表面、Logo 局部UV或烫金、产品名轻度UV/击凸、背景轻纹理，小号文字不做重工艺处理。
- 避免空洞的形容词。每条推荐必须明确指出：材质、工艺、区域、目的和风险。

当用户说「直接生成效果图」「按这个做」，并提供了标注区域、材质和输出位置时：
- 在简要总结图片角色/区域/材质后，执行面板拆解 → 图生图生成。
- **禁止写 Python 脚本做本地合成。** 只能走图像模型生成路径。
- 除非物理尺寸或图片角色存在不安全/歧义的情况，否则不要停下来长篇询问。

## 物理逻辑检查

对于圆柱形或环绕式标签：
- 若任务依赖正面视角逻辑，且用户未提供瓶身直径和标签宽度，应主动询问。
- 周长计算公式：`直径 × 3.1416`。
- 若仅需正面渲染图，只展示物理上可见的正面部分；侧面/背面的信息面板应翻转到视野之外，仅可能露出边缘小条。

对于方形/矩形瓶或盒子：
- 若标签贴在平面上，保持正面标签为平面。
- 遵循可见侧面的透视关系；不得让工艺效果跨越硬折角，除非画面内容确实环绕包裹。

对于工艺标注图：
- 红色矩形框、彩色线框、遮罩和注释意为「在此处施加工艺」，而非「画出这些标记」。
- 若标注覆盖了较大区域，将工艺施加在该区域内的画面元素上，而非整个矩形背景，除非用户明确要求整版上光。

对于刀模图：
- 不得在压痕/折叠线、糊口、切割边缘或窄小插舌处施加厚重工艺层，除非明确有意如此。
- 将工艺区域保持在安全边距内；避免工艺层过于靠近切割线或折叠线。
- 刀模线条、折叠线、切割线、面板轮廓和套准标记不得作为印刷线条出现在最终 3D 包装上。
- 若任务可能进入印刷生产环节，提醒：出血通常为 3mm，文字应转曲，黑色小字通常应为单黑，烫金/UV/击凸/白墨需要单独分层。

对于 SVG/矢量刀模图：
- 解析 `viewBox` 或画板尺寸，并在可见时报告面板尺寸/比例。
- 利用线条坐标推断面板网格结构，但将其归类为非印刷引导数据。
- **必须**渲染高分辨率参考图（2x），用于后续面板拆分。
- 创建面板标注图（绿色线框 + 命名），仅作几何引导，禁止标注标记出现在最终图中。
- **必须**在图像提示词中包含精确数值约束，如「每格面板约 217×214pt，折叠后正面接近正方形 1:1」。
- 仍需提醒：图像生成只能大致还原画面/文字；严格的 1:1 生产打样需要真正的 3D/包装 mockup 软件或确定性纹理映射。

## 盒型结构视觉规范（生成 3D 效果图时必须遵守）

**图像模型容易混淆不同盒型的 3D 外观。生成提示词时必须包含目标盒型的明确结构描述。**

### 天地盖硬纸板礼盒（Rigid Lid & Base Gift Box）
- 两个独立硬质部件：**盖子**（浅盒子从顶部套下，覆盖约 1/3 高度）+ **底座**（较深托盘）
- 外观特征：整体矩形块体，盖/底之间水平自然缝隙（阴影/纸边，非印刷线）
- 盒体厚实硬挺，边缘可见 1.5-3mm 灰板截面
- 表面平整，无外露插舌、无糊口、无自锁底
- **禁止特征**：外露折翼/插舌、锁底结构、瓦楞纸、薄卡纸
- **若只有盖子平面稿**：只生成盖子，底部边缘为灰板截面自然终断

### 反向插锁盒（Reverse Tuck-end Box）
- 单片卡纸折叠结构，上下两端插舌方向相反
- 上端插舌从前面向后面折叠插入，下端插舌从后向前折叠插入
- 盒体为矩形，侧面可见折叠边和插舌边缘
- 材质为卡纸（非灰板），比天地盖薄但硬挺有型
- 通常为紧凑小盒，适合食品/化妆品小包装
- **禁止特征**：灰板厚度感、天地盖的盖/底水平接缝、自锁底结构

### 飞机盒（Crash-lock Bottom / Mailer Box）
- 单片折叠结构，底部有交错自锁底翼，顶部有可翻开的插舌翼
- 外观特征：顶部可见插入式折舌，底部可见十字交错的锁底结构
- 材质较薄，通常为瓦楞纸或单层卡纸

### 插口盒（Tuck-end Box）
- 单片卡纸折叠，两端有插入式折舌（同一方向）
- 外观特征：端面可见插入的折舌边缘

### 抽屉盒（Drawer / Slide Box）
- 外套+内抽屉两件式，内盒从侧面滑出

### 书型盒（Book-style Box）
- 类似精装书封面，盒盖可翻开，通常有磁吸翻盖

**生成提示词时必须根据识别到的盒型，在 prompt 中加入对应的结构描述和禁止特征。不得省略。**

## 工艺规划默认值

当用户未提供工艺映射时：
- Logo/品牌标识：根据基材选择局部 UV、烫金或素击凸。
- 主插图/图标/图案：局部 UV 或触感 UV，实现局部提亮。
- 产品名/标题：克制的 UV/烫金，仅在文字可被保留的前提下使用。
- 背景：哑光/软触感/特种纸/PET 纹理；避免繁杂的效果。
- 法规文字/营养成分/小号文字：不做处理，保持可读性。

## 特殊材质与工艺速查

| 材质/工艺 | 视觉效果 | 适用场景 |
|---|---|---|
| 镭射银卡纸 | 基材自带全息彩虹光泽，银色金属底色 | 食品/化妆品小盒，吸睛高端 |
| 逆向 UV | 大面积哑光触感面 + 局部亮光 UV 对比 | 卡纸折叠盒，增加触觉层次 |
| 局部露银 | 特定区域不印油墨，露出银卡纸底色 | 配合镭射银卡纸使用，金属细节 |
| 银龙 PET | 银色拉丝/镜面 PET 薄膜，金属光泽 | 瓶标、不干胶标签 |
| 触感膜 | 哑面天鹅绒般触感，低反光 | 天地盖礼盒、高端盒型 |
| 珠光特种纸 | 自带珠光微粒纹理，角度变换光泽 | 礼盒裱面，新春/节庆包装 |
